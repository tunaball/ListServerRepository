Here's the solution to the question.

The import command takes the POSIX path as it's argument.

Unfortunately, if the command fails, no error message is given.

The mbox file must be named "mbox" and it must be in a folder that  
has the extension ".mbox", eg. Burst.mbox/mbox

Additionally, line endings must be ASCII character 10, not 13.
