Interesting. Mac OS X 10.4.8?

I had the same symptoms Thursday last week. Start Mail, it begins to  
download messages (spinning disk beside accounts in Mail) and then,  
after a while, poof. Mail gone. Every time. At first I thought a  
corrupt email on the server - but eventually ruled that out. Rebuilt  
the prefs file. No good. Crash log showed a fairly uninformative  
generic memory error. Trashing everything in ~/Library/Mail/ which  
wasn't a folder (I kept "Signatures.plist" though - too lazy to have  
to rebuild them unless I have to), fixed the problem.

It's my hypothesis that the issue was caused by a corrupt  
MessageRules.plist (it's only when the rules are applied that Mail  
blows up). Try moving "MessageRules.plist" and  
"MessageRules.plist.backup" to the desktop and then starting Mail.  
Does that fix the issue? If it does, you'll have to rebuild your  
rules - but if they're anything like mine you've got a few years of  
cruft in there, so no bad thing.

Let me know if this helps.
