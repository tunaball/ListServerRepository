I had a play with this the other day and found that if you close the 
widgets they don't consume memory any more.

When you invoke Dashboard, press the + in the bottom left corner. Xes 
should appear on all active widgets. Click each X to close them and you'll 
see them disappear from Activity Monitor.

On 9/26/05 4:26 PM, Robert Nicholson <robert at elastica.com> wrote:

> even when Dashboard is not active. Activity Monitor is showing that my
> Dashboard widgets are using Real  Memory even when Dashboard isn't 
active.

Once you invoke Dashboard for the first time after you log in, it is 
always
active until you log out or quit the Dock. The running widgets will 
consume
memory (and, on rare occasions, CPU) regardless of whether the Dashboard
itself is visible or not.
