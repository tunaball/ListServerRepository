On 9/26/05 4:26 PM, Robert Nicholson <robert at elastica.com> wrote:

> even when Dashboard is not active. Activity Monitor is showing that my
> Dashboard widgets are using Real  Memory even when Dashboard isn't active.

Once you invoke Dashboard for the first time after you log in, it is always
active until you log out or quit the Dock. The running widgets will consume
memory (and, on rare occasions, CPU) regardless of whether the Dashboard
itself is visible or not.
