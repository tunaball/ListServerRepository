> Encoding for Windows is supposed to strip away the resource fork.

Yes and no.

Yes, if you encode for Windows then the attached file holds only a  
data fork, because the concept of a resource fork doesn't work with  
plain attachments.

But no, the resource fork isn't gone. The content of the resource  
fork has been added as a specially marked part of the data fork of  
the attachment. So if people decode at the other side they still have  
both.
