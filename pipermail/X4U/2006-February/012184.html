On 2/11/06 12:09 PM, William Scammell wrote:

> the one thing I've noticed, is that the 80GB Maxtor does not indicate
> an osX "mount point" nor os9 "driver" (via System Profiler output
> seen below) compared to the working 300GB Maxtor. I'm guessing this
> is the problem (or a sub-board in the case). So, *how* do I force osX
> to mount this infernal thang?

If OS X isn't seeing the mount point, you can't.

You can use something like DataRescueII from ProSoft Engineering, which
*will* read data from non-mounting drives. The demo lets you recover one
file so you can determine whether it works for you or not.

I have an external Maxtor FW, and it loses its mount point frequently. The
only solution I've found is to reformat the disk.
