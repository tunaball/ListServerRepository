At 9:28 AM -0600, 11/23/07, Linda wrote:

>So, please tell us, how do you test for bad RAM if you don't hang 
>onto the original?

Users always have at least two DIMM / SIMM. Remove one, then see if 
the problem goes away. Then test the other. The two don't go bad at 
the same time. RAM typically lasts a lifetime and rarely goes bad but 
being able to run the system on one stick is surely enough a test for 
anyone.
