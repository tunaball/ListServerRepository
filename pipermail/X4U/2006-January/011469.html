Hi!

Just had this happen to me while reinstalling 10.3.9 on a hard drive 
that could not boot.

What I was doing, in the interest of saving time, was using software 
update to see what needed to be updated, then using my saved package 
files located on other drives, using them to update the system 
software.

When done, i ran disk utility and got the error you described.

Got it to work by putting all the packages used for the update in the 
package folder in the library where they belong, and where they would 
have been had I downloaded them, making sure the receipts were in the 
receipts folder, rebooted using this drive, and running repair 
permissions on that drive.

It worked for me.
