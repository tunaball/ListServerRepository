I solved a password problem recently and it might have been just after 10.3.8 but I'm really not sure it didn't occur with the introduction of Panther.

I needed to become root with a su command and not just the sudo which I perform regularly. Terminal simply said "sorry" and I did not forget my password which is different from my administrator password for use with sudo and the like.

The solution was to, in NetInfo Manager, use the menu to turn off root access. It asked for a password but responded to a simple return and root access was turned off.

After that I was able to restart root access and the machine said I have to provide a password. I put my old one back in and all was well.

An explanation, without a proof, is that somewhere between my root accesses OS neXt forgot my password and set it to blank without disabling root access. In my case that's somewhere between the first install of Panther and 10.3.8. I can't remember the last time I needed anything more than sudo.
