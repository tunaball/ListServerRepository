I'm sure you're frustrated, but you should remember two things:

First, it was your Macintosh that locked up and needed rebooting.

Second, what was corrupted and wouldn't work afterwards was information
ordinarily handled by a Macintosh program (the Windows emulator). So,
perhaps VPC isn't as robust as it should be. I don't think one can cite this
as yet another example of how the Mac OS is more robust than Windows. It's
entirely possible that Intel Hardware would have handled this crash with no
problems whatsoever.
