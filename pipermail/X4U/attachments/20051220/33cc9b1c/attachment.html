You've only got data transfer through one of the pipes. I don't know
which one is chosen by default and its likely that you can specify one
for a particular app (FTP, etc.) but you aren't going to be splitting
the workload between them.<br>
<br>
On servers you'll have two network interfaces (NICs) sharing a single
IP. This is done for load balancing and/or fail over redundancy. But
generally, these server NICs have circuitry that most consumer grade
NICs don't. With consumer grade NICs you generally need third-party
utilities to team the NICs, but I don't see a purpose for going to the
trouble and expense for the scenario you describe.<br>
