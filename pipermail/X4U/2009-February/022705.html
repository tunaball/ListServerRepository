On 2/4/09 at 6:08 AM -0500, Neil Laubenthal wrote in a message entitled
"Re: [X4U] Remote Access to Mobile Laptop - Any Solution":

>I can think of a couple easy options . . . although I'm sure with more thought there are likely more.
>
>TimbuktuPro - you can set the laptop to register itself with the Timbuktu server when it comes online and it will show up on the World List (I think that's what it's called) in the Timbuktu Pro console. This works fine but I think it might have needed an outgoing port opened.

Yes, I can get the WAN IP number that the laptop is running on, but without a standard or alternate Timbuktu port opened on the router, I can't get the the machine, right?

>LogMeIn - If you install LogMein (www.logmein.com) on the laptop it will login to their server via port 443 (which is always open on a NAT router unless it was specifically closed which will break https). LogMein is cool . . . it establishes a three way VPN between the browser on your support machine and the client. It ranges from free to a few bucks a month depending on which features you need/want.

My understanding is that LogMeIn remote would do it (at $129/month), assuming that the administrative computer is running Windoze. (The client computer can be a Mac.)

>BacktoMyMac is also a possibility; but this is trickier since you need a PnP capability in the NAT router.

I haven't check BackToMyMac out yet.
