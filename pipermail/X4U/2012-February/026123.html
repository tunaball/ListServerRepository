Today I've just got my first Lion host on my network, replacing my 
wife's 12" iBook with a new 13" MBP. When moving files from the old 
to the new that Migration Assistant didn't copy it worked just like 
I'm used to between any other two OS X Macs.

When you connect to the other Mac, if you do so as the user that owns 
the files and folders you want to access, there should be no problem. 
Copying a file across and getting an empty file means that the 
destination folder is writable by the user doing it, so the file is 
created, but the file gets permissions that don't allow the user to 
write the contents into it. We need to know more to diagnose the 
problem. Have you made any changes to try to help the situation?
