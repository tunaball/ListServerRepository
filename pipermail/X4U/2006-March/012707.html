On 6 Feb 2006, at 13:35, Daly Jessup wrote:
> ...
> I did go to grc.com and learned that port 80 is in "stealth" mode.  
> Would that be caused by the ISP blocking it?

This would certainly indicate that your webserver is not accessible  
from the outside world.

> If I have to change the port for the web server, can someone  
> explain how to do that?

The easy way - if your router allows it - is to forward incoming  
requests on (say) port 8888 of the WAN address to port 80 on the  
machine inside the LAN. Otherwise edit /etc/httpd/httpd.conf:

	...
	#
	# Listen: Allows you to bind Apache to specific IP addresses and/or
	# ports, instead of the default. See also the <VirtualHost>
	# directive.
	#
	#Listen 3000
	#Listen 12.34.56.78:80
	...
