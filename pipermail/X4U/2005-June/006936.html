On 6/9/05, DZ-Jay <dz at caribe.net> wrote:
> Hello:
>         I know this will sound stupid for you, but I'm an ex-Windowze user, so
> when considering executables, I think of registry hives and DLL hells.
> My question is, if I want to organize my Applications folder, can I
> just move the applications (post-install) into sub-directories, and
> then update any links/aliases, or would that completely break the
> application and its dependecies?
>

Yup, 99% of the time you can put applications anywhere you like. And
guess what...you don't even have to update your aliases. Mac OS X is
smart enough to find the file again if you use the Finder to move it.

The only time that you may have difficulty is if you relocate Apple
applications (iCal, iSync, utilities). When you run software update
Apple expects to find those apps in the Applications folder and they
may not update if they're not found there. I don't know if this issue
has been corrected already.
