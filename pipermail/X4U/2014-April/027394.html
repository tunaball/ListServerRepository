>Folks;
>	I've run across a problem new to me:
>		I've a Bellsouth.net sub account that has quit getting new messages in mail under 10.7.5.
>	The account gets new mail, and on other devices (iPad & iPhone with i OS 7.0.x), the new mail comes thru fine; but on my MacBook Pro running 10.7.5 in mail, it won't get the new mail.  I can see it getting the count of the new mail, then it dies.  No new mail gotten.
>	ANY thoughts????

The initial request for a count is typical of a POP3 mail connection.

Apple's "mail" app pushes hard in the direction IMAP as opposed to POP3.  I wouldn't be surprised to find that Applemail is making IMAP like requests on a POP3 server.  Are you sure you have the email URL port set correctly?

Providing IMAP storage for previously downloaded mail on the server can be a problem on the server side that doesn't exist when only POP requests are honored and each message is deleted after a successful download.
