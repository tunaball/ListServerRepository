On 05/25/05, revDAVE <coolcat at hostalive.com> wrote:
>
> Currently I have OS 9.22 on its own partition - all loaded up and working
> well... ( fully bootable on my older dual 1.25 ...)
>
> -  I need to transfer this to a different partition on a different drive
>
> -  what is the best way to transfer this entire partition with 9.22 to
> another blank partition -  and keep all the system related aliases working
> correctly?

While in the Finder, select the partition with OS 9, and drag it to the new
partition. That's all. All aliases, etc should be duplicated. You don't
have to change it's name or do anything else. Boot into the new partition
and check things out.

>
> -  I do have Carbon Copy Cloner ... Would this be the best app for this
> purpose to keep the system related aliases working correctly? ... Or is
> there another way to do this?

CCC is inappropriate for duping OS 9; it's for duping OS X.


> -  I know that simply dragging the entire system to a different partition
> could screw up the alias relationships ... Right?

Wrong.

>
> Q: -  is it true that if I did drag the entire system 9.22 to a different
> partition - and then named that partition exactly like the old partition
> name ( and changed the old one to something else) -   Would this preserve
> the system alias relationships?

Not required.
