My understanding is that port 587 is reserved for authenticated SMTP,  
so it would be unusual to see this port blocked. In fact, port 587 is  
the preferred port for users to submit messages into the network.  
There is an RFC that describes the differences between message  
transfer between message transfer agents (on port 25) and message  
submission by message user agents (on port 587). Here's the RFC which  
has more information that I could digest <http://www.faqs.org/rfcs/ 
rfc2476.html>
