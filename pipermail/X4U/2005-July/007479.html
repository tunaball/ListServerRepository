Spotlight is not accessing the actual files while searching. Rather it depends on an index which it has created in background. The index probably contains "words" and retains no information about their order. Well, perhaps it pays attention to some specific combinations that it found important at  indexing time.

You would be very unhappy with the speed if Spotlight was really examining every file as you search. You would also be unhappy about the size of the index it it kept track of pairs, and triples, of words.

And while I'm at it, does anyone know how to put the Spotlight index for a CD-ROM about to be created on said CD-ROM? Sherlock was pretty helpful that way.
