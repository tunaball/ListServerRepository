On Jul 3, 2005, at 12:58, Dennis B. Swaney wrote:

> if Safari doesn't use them, why are they in Safari?

Who says the plug-ins are "in Safari"?

"On Mac OS X [browser] plug-ins should only be placed in the 
/Library/Internet Plug-Ins directory." (tech note TN2020)

That's where they are, and that's where all browsers are supposed to 
look for plug-ins.

> Can they be deleted?

Not a bad idea; not to delete them, but to remove them. For instance, 
create a new folder, /Library/Internet Plug-Ins (Disabled), and move 
there Java Applet Plugin Enabler and Java Applet.plugin, but don't 
touch JavaPluginCocoa.bundle. Then quit and relaunch Safari; this time, 
it should report Java  1.4.2, and this might fix the OP's problem.
