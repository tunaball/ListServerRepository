Macaroni is NOT a total cron replacement since it has no provision 
(unlike cron) to say WHEN (on a designated day) the command will run 
(it runs whenever it decides to on the designated day). Also, it can 
not run more than once a day (you'd need to define 24 separate jobs 
to run hourly and there is still the issue of being unable to say 
what time each of the jobs will run). The only advantage it has over 
CRON is that it can run jobs that should have run while the computer 
was powered off and can be told to run a job every time you boot the 
computer.
