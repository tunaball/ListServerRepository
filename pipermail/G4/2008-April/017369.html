On 7 Apr, 2008, at 2:01 PM, Robert A. Rosenberg wrote:

> Macaroni is NOT a total cron replacement since it has no provision  
> (unlike cron) to say WHEN (on a designated day) the command will run  
> (it runs whenever it decides to on the designated day)

  You can use the built-in scheduling feature to only permit the  
program to run AFTER A DESIGNATED PERIOD OF IDLENESS.

That makes it a no-brainer if you allow your machine to sit idle for  
the specified length of time on the day you have selected and let  
Macaroni run the CRON task(s) you choose. At that point, you can shut  
down or put the machine to sleep as you wish and Macaroni won't run  
the tasks again until the scheduled day comes around again.

Hope this helps someone.
