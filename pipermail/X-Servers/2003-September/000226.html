I'm sure you've tested access to the ftp server from the same client OS and client ftp app from the local lan -- right???
If it is working from the LAN and not from outside the firewall (again, using the same client and os), then find out if your data packets are even reaching the ftp server and a response is coming back.  Do this by simply telneting to the ftp server from a terminal app or shell or whatever type:

> telnet ftpsrvr.domain.com 21

and you should get some type of response back from the ftp server.  The built-in ftp server sends a prompt something like this:

[dalton at spies dalton]$ telnet osx 21
Trying 192.168.1.32...
Connected to osx.
Escape character is '^]'.
220 ftp.domain.com FTP server (lukemftpd 1.1) ready.
quit
221 Thank you for using the FTP service on ftp.domain.com.

If you get a prompt from telnet server, then I'd say it is a client-2-server incompatibility and it shouldn't have worked from the local LAN either.
If you don't, then you have a firewall issue....

Good luck
