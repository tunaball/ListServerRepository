Has anyone had permission problems when saving to a Tiger or Leopard Server? We are a small publishing company and the workstations with Leopard or Snow Leopard will occasionally have problems saving changes to a document. The document will end up needing to be saved under a different name. Investigating the problem file shows the owner and group permissions have changed to "unknown". Using the Propagating Permissions option from Server Admin will temporarily fix any of the documents with this problem. My Tiger clients are not having this problem. Switching from the old POSIX style permissions to ACL style permissions as a possible fix didn't work. The clients sessions with the servers (two old G5 towers, one running Tiger Server and one running Leopard Server) are via AFP.
