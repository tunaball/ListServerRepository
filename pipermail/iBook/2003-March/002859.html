You can either trash them, or move them to a temp folder.
Help is also slow on my machine. I think that that is inherent to it's 
design.
When you install an application by just copying it in the /Application/ 
folder, it's help file will show up in Help. The help files for 
application bundles are inside the localised resource directories 
inside the bundle. So I think that help scans the application bundles 
during startup causing the slow performance, I might be wrong though.
