/sbin is probably in your path when you boot into single-user mode. The
backslash is an escape character, but apparently the shell found no proper
character to escape after it, so it was ignored.

That said, just typing things in single-user mode without knowing what
you're doing isn't a very good idea. You are root when you are in
single-user mode, and hence can wreak havoc on the file system if you don't
watch out.
