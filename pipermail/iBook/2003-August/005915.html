The trouble is that mail messages are kept in a mailbox, which is a 
file in your library called "accountname.mbox" so the message itself 
can't have different permissions than the file - if you can access and 
delete other mail in that account then the permissions for the mailbox 
itself are fine.

What I would suspect is that the mbox file is corrupt and it's causing 
problems deleting the particular message.

Try all the usual suspects - repair your disk permissions and do a disk 
first aid check from the boot CD on the master volume. Also let 
DiskWarrior have a go at it if you have it.

Joe
