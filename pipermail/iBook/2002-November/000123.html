>Brian D. Silverio at 11/25/02 (maybe):
>So the answer to your question is:  fsck is run at boot
>only if the system crashed.

Brian, you're correct.  fsck is run at boot time when the OS encounters
a volume that was not properly unmounted that last time the machine was
powered down.  So, a system crash will do it, pressing CMD-CTRL-POWER
will do it -- basically anything short of letting the machine shut
itself off will do it.
