I have found simply using Graphic Converter and a date structured  
file layout works well.  I set GC to make large previews and then  
when I visit a folder all the photos are displayed in adequate size.   
GC is a phenomenal program, but the interface is perhaps better for  
experienced users.

The catch is that I use a unix program (with a shell (bash) function)  
'jhead' to convert the file name of my digital photos to "year month  
day hour second" (no spaced) and a simple Perl script I wrote to file  
the photographs (and create the required directories).

What I'm left with is a tree of folders Year/Month/Day  each  
populated with photo's from that day.

I also have Perl scripts to generate input from the photo directories  
into an SQL database (PostgrSQL) and viewable in Filemaker but that  
is overkill.

On the commercial front I would try iView Media' $50, the stepchild  
of 'iView Media Pro'.
