Naaah. . .

OS neXt users of the UNIX tools need to understand about $HOME/.Mac_OSX/environment.plist.

It's a much more reliable way to set up $PATH and other environment variables that you REALLY want to get right. Other ways of getting started depend too much on your choice of shell when you even can. You also get forced into bash when you execute a script from a text editor with a service or use Do shell script in AppleScript.

If the OP had an environment.plist file he needs to go there to make some of the changes required, on particular being sure his $PATH gets the right bash if no #! line is used.
