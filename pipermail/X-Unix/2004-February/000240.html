Thought of the simplest, ultimate solution to this last night.
Have a cronjob (AND a startup script, and maybe a wakeup script) that 
does this:

"curl"'s a nonexistent yet unique URL (such as 
http://lostlaptop.dyndns.org/thisisyourlaptopcalling) from a web server 
whose logs you have access to. If you use a dyndns URL, you can then 
point the requests to the IP of any web-serving machine available to 
you and then just wait and examine the logfiles. No dealing with SMTP 
issues, no emailing at all actually, and the IP in the logs will be the 
router's IP if it's behind one.
