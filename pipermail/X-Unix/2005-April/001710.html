When using wildcards (a.k.a. "file globbing") in find terms 
like -name you want to quote the pattern, like saying

find . -name '*.txt'

This is to prevent the shell from expanding the wildcards
too soon in a different context. (Or in the case of -regex
completely misinterpreting them)

(Using the results of a find, you need to be careful on Max OS
that you properly quote file names containing spaces and other
unusual characters.)

It might also be worth considering find piped into egrep
with and without -v, as you can use it to do regular expressions
on the file path to match and reject things. Sometimes
this is easier than following the quirks of find (which
vary a lot across Unix flavors, etc.)

find . -type f | egrep -i 'match' | egrep -v '(ignore|skip)'
