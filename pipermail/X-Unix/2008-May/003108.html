That's probably your best bet.  With the proper developer tools, this
should be relatively trivial.  Also, if you have an old version of Mac OS X
around, you could probably simply copy the binary to your system, to an
alternate location such as /usr/local/bin, or /opt/bin/, so as not to
interfere with the system installed version of OpenSSH.  If you run into
library problems, you can probably symlink the newer libs to a name
matching what the old binary is looking for.  It's kind of a hack, but
works in a pinch.
