To further complete what Eric said, I'd like to mention that you need to
comment out the line defining extension_dir as "./" in php.ini, by adding a
semicolon in front of it.

Then, simply add the name of the module which I assume you compiled by
running phpize in the module directory, and then installed with make
install. (most likely pgsql.o)

You may then restart apache running 'sudo apachectl restart'.

I just added GD2 support to the default Apple PHP install, and it worked
fine.

'phpize' should work out the details for you, such as the module location.

Hope this help.
