We are testing Panther (10.3.2) for deployment and have hit a snag that we feel may be LDAP related but 
are not sure.  The symptom is that the Finder believes the user is root (or admin at least) while the 
terminal correctly identifies the user including group ID.  The user also cannot log out and must shut the 
machine down or force quit the Finder which rectifies the situation and then the Finder is happy and no 
longer confused.  The issue seems to be limited to network users and is intermittent.  We are using 
Netscape 5.2 LDAP service on Solaris 9. We have ported Apple's schema extensions to the Netscape 
LDAP service and are wondering if anyone has had any similar issues or has any thoughts on the matter.
