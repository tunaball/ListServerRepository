Apple has changed its recommended naming of preference files to  
conform to standards which can make it a bit more difficult to find  
what we're looking for. If the pref file can't be found by looking  
for the program's name, then look for it by the developer's URL -  
backwards, then the program name:

in other words, if the program is named Bar and the developers  
website is http://www.foo.org look for the pref file like this:

   org.foo.bar.plist

so you are looking for

   org.mozilla.firefox.plist

Given that this can be a pain...is the URL a com, net, org??? I just  
tend to do a search inside the preferences folder on the filename.
