> That's a downer. How come? Is it a UNIX thing?

No.  It is a limitation in the ASIC used in the disk controller.   You 
shave a few dollars off the cost if you use mature technology with disk 
addressing limitations.

The controller may be able to use an oversize disk by pretending that 
it is within its limits.  When you upgrade to a more capable disk 
controller, you need to drop that pretence.  The way that is done is by 
reformatting the drive to recover the full extent.
