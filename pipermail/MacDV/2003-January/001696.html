On Thursday, January 23, 2003, at 09:27 PM, Thubten Kunga wrote:

> Doesn't the iDVD 2.1 burn in OSX run in the background too?

It runs in the background, but if it doesn't get the CPU time it needs 
because of your foreground process, you'll get a buffer under-run error 
and the burn will be ruined (the data is being compressed dynamically 
and it can't find the "restart" point if the CPU doesn't keep up). When 
you're burning an image, that's not an issue.
