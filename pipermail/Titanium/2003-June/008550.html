John,

Wine stands for 'Wine Is Not an Emulator' and that is correct. I mean 
that it isn't an emulator. The way it works (as I heard) is that they 
take the Windows API calls and translate them to the kernel calls for 
Linux. The kernel then passes them to the processor. There is no 
translation done, all code is intel x86 specific, neither is there a 
virtual machine a la VPC that emulates an intel x86 processor.

So yeah, I know something about running Wine on OS X, namely you can't 
run it.

How the Bochs project seems to be able to run an IA 32 chip on Mac OS 
X. See http://bochs.sourceforge.net/
