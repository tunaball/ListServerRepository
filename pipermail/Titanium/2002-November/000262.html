No workarounds that I know of.  I've been running version 4.6.3 of the 
HP driver software on 10.2.1 and 10.2.2 and it works equally bad on 
both.  The previous 4.6.1 release seems slightly less problematic on my 
wife's system running 10.1.5.

All in all the HP software leaves much to be desired.  If you need to 
unhang your system after printing, open terminal, run top and look for 
the HP Communicator task that will be utilizing > 90% of CPU resources, 
then kill the task.  I've not found a clean way of restarting the task 
without restarting the system, but at least you can postpone that until 
you want to print again.  YMMV and all that...
