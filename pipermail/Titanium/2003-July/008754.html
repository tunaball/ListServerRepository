I'd guess you're talking about ._<name of file> that shows up when you  
transfer files to a Windows machine, but you never see on the mac.   
They are resource files that the mac uses but windows machines do not.   
I posted a link on MacOSXhints about this topic and the comments go a  
little further:
http://www.macosxhints.com/article.php?story=20020217094739452
My solution was simply to use the UNIX cp command to copy the files, as  
it does not copy the resources.  This can be done from the terminal or  
through a GUI front end to the command.
There is also another hint with comments that might be helpful here:
http://www.macosxhints.com/ 
article.php?story=20021118060850652&query=%5C%22._%5C%22
