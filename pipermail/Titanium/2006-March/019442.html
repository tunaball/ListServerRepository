I assumed that you are all backed up.
I had similar problems with bad RAM and then later with a unseated RAM 
chip.  Both of these can cause file structures, etc to become 
irretrievably corrupt and will cause the repeated kernel panic and even 
a seeming harddrive failure.

Reseat the RAM first and then if you still have trouble have it checked.

None of the standard utilities indicated anything was wrong(even 
Techtool).
