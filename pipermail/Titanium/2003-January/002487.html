And so to summarize the test I've just performed,

Eudora successfully sends messages to the list without breaking long 
urls while not using the < or >. Eudora also displays the urls as 
links while composing the message.

Entourage will cause the message to break unless the user encloses 
the url in < and >. Entourage will display the url as a link while 
composing.

Apple's Mail will successfully send a message without the < and >. 
Mail will not display the url as a link prior to sending. (In fact, 
the url breaks in the composition window weirdly and you have no idea 
what the heck you are going to get.)

Once again, Eudora wins.  :-)
