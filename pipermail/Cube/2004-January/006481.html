> 1536mb of RAM which is normally split 168 used and the rest (maths in
> public should be avoided at all costs) as unused.  However over time
> that used bit creeps up to about 356 used.  Does anyone know why that

Sounds like a memory leak. Some poorly written applications continually
request more and more memory which eventually bogs down the OS. Rebooting
resets it back to normal, until it starts happening again.

You don't mention what OS you're running which makes it hard to advise, but
you can use Process Viewer or something similar to see what's up with each
application.
