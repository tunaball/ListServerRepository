On 2006.07.29, at 06:07, Andrew wrote:

> Hello all
> I've got a 450 MHz cube running 10.3.9 (panther).
> The cube sleeps overnight so I've been running Onyx (ver. 1.5.2)  
> weekly.
> In the last month or so I've been getting error messages.
> A look at the log shows an "Error detected while rebuilding the  
> whatis database of X11R6"
> Repairing permissions with Disk Utility makes no difference.
>
> Here's the error log:
>
> - Verifying and repairing permissions (Mac OS X)
> - Optimizing the System
> - Executing the Daily script
> - Executing the Weekly script
> - Executing the Monthly script
> - Correcting permissions of the file: install.log
> - Correcting permissions of the file: wtmp
> - Error detected while rebuilding the whatis database of X11R6.
> - Updating the LaunchServices database

He most probably receives some kind of problem while rebuilding  
whatis database. The database is rebuilt by running "makewhatis"  
command, so you may see what went wrong by manually running this  
command in the terminal.
My suggestion would be to turn off whatis database rebuilding as it  
is, dare I say, irrelevant.
