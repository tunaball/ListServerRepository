try to run the command 'mount -a', which ought to mount your 
filesystems with proper permissions.  After that, you can run the 'ln 
-s /private/tmp /tmp' command described earlier, and it should work.  
If you get some funky error message with 'mount -a', try running 'fsck 
-y' and then run 'mount -a' again.

if that still doesn't work, you could try this (not sure if this is 
implemented correctly in osx):

# df -k

look for the first line to say something like /dev/disk0s9 -- my 
computer says disk0s9, yours may say something different.  Other 
columns say devfs, fdesc, etc.  You don't care about those right now.  
Force a mount of it with read/write priveleges.  Don't cut & paste 
here, replace disk0s9 with whatever your df -k output is.

# mount -w -t hfs /dev/disk0s9 /
