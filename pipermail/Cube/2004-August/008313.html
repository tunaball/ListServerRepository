It is my understanding that Disk Utility's Repair Permissions looks in 
/Library/Receipts to determine proper permissions. Any later installed 
packages should have created a .bom file there.

See http://docs.info.apple.com/article.html?artnum=106712
and
http://docs.info.apple.com/article.html?artnum=25751

>> Repaired permissions using installation CD last week but problem has started
>> again a couple of days later.
>
> DO NOT REPAIR PERMISSIONS FROM THE INSTALLATION CD
>
> More than likely, your OS has been updated since you installed it. The newer
> parts may need their permissions set differently than they did at the time
> your installation CD was current. For all you know, you might be setting
> certain permissions to the wrong values by repairing then from your install
> CD.
