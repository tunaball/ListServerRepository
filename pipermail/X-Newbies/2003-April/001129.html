Usually, that time is spent looking for a server from which to NetBoot. 
Since a NetBoot Server could potentially be busy, there is a longer 
timeout for that search.

You should be able to do Cmd-. (Command-period) to cancel it from 
looking for the network disk, to reduce that timeout.

There isn't much space for fancy interface in Open Firmware -- it's a 
rather small amount of code with a small memory footprint.
