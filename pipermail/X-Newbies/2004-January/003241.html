I did have trouble with Application Enhancer at first, but then 
Unsanity updated it and all has been well ever since.

Audio Hijack Pro, in addition to working as a passive receiver of 
streaming audio, can also be used as a server of streaming audio. This 
is why it wants to install APE -- so that it can then install its own 
Server start-up items.

If you do not plan to use this aspect of Audio HiJack Pro, you can 
safely disable APE and the start-up Audio Hijack Server item. It will 
not affect the running of AHP when it comes to recording audio streams.
