Hi Judy,

Have you tried logging in with a "clean" user folder?  If not create a 
new user, name and password of your choice, log in as this user and see 
if TextEdit will launch.  If it does launch with a clean user account, 
you will know that something is messed up, or causing a conflict in 
your regular user account.

Also just some things you may have tried or not.  Repair permissions, 
first aid from the install disk, or other repair utility.  Log in with 
the shift key depressed, log in, in single user mode to run fsck -y.
