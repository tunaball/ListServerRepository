Richard wrote:
> Running XL under 9.1 on an 8500 with 144Meg RAM one is continually
> beset with  -

> 'Not enough memory' messages from within XL and sometimes other
> messages of a non co-operative nature 
>
> Adding more memory to XL has done nothing [even up to 30Meg+], nor
> has using Virtual memory with an extra meg or an extra lotsa Meg...

Does the file have a macro or not? That can make a difference. If you have
other specific error messages, it helps if you can write them out exactly
as they appear. 

One thing you can try that I found in the MS knowledge base: it seems that
Excel really wants to have its memory increased in even multiples of the
minimum. The example they gave is that if the minimum RAM was 6344,
you should increase it to 12688 or another multiple and not some arbitrary
value in between. You can find that info here:
< http://support.microsoft.com/default.aspx?scid=kb;en-us;268410> 

Again if you see other error messages, write them exactly as they appear
and that may help to find the cause.
