Avoiding use of the Scratch That command and instead selecting text 
followed by saying Delete This  appears to minimize the likelihood of 
ViaVoice failing to produce text.  It isn't necessary to run audio 
setup each time.  Unless you have removed the headset and then put it 
back on--in which case Audio setup serves to check that you have the 
microphone in the proper position.  For  me when ViaVoice fails to 
produce text I can save manually (including the speech session data), 
exit ViaVoice, restart ViaVoice, load the saved SPAD file and 
continue right where I left off.
